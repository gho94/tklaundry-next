## 1. 네이밍 원칙

- **레거시(기존 C#)와 구분**이 되어야 함
- Java·Flutter 각각의 **언어 관례**를 따름
  - Java(Gradle): `kebab-case` (예: `my-project`)
  - Flutter: `snake_case` (예: `my_project`) — pub.dev 규칙 강제
- 모노레포 폴더명과 실제 프로젝트 내부명을 **일치**시킴
- 너무 길지 않게, 나중에 확장해도 어색하지 않게

## 4. 각 프로젝트 초기 설정값

### Java (Spring Boot + Gradle)

```
프로젝트명(rootProject.name): tklaundry-api
그룹(group):                  com.tklaundry
패키지 기본:                  com.tklaundry.api
artifact:                     tklaundry-api
빌드 산출물:                  tklaundry-api.jar
```

`settings.gradle`
```groovy
rootProject.name = 'tklaundry-api'
```

`build.gradle` (또는 `build.gradle.kts`)
```groovy
group = 'com.tklaundry'
version = '0.1.0'
```

**패키지 구조 예시**
```
com.tklaundry.api
├── TkLaundryApiApplication.java
├── order/
│   ├── OrderController.java
│   ├── OrderService.java
│   └── OrderRepository.java
├── common/
│   ├── response/
│   └── exception/
└── config/
```

---

### Flutter (Windows Desktop)

```
flutter create --project-name tklaundry_app --platforms windows app/tklaundry_app
```

`pubspec.yaml`
```yaml
name: tklaundry_app
description: TKLaundry 차세대 관리자 앱 (Windows)
version: 0.1.0+1
```

`flutter run -d windows` 시 타이틀바에 표시되는 이름:  
→ `windows/runner/main.cpp` 또는 `windows/CMakeLists.txt` 의 `BINARY_NAME` 값

```cmake
# windows/CMakeLists.txt
set(BINARY_NAME "TKLaundry")
```

**폴더 구조 예시**
```
lib/
├── main.dart
├── app/
│   ├── router.dart
│   └── theme.dart
├── features/
│   ├── order/
│   │   ├── order_list_page.dart
│   │   ├── order_detail_page.dart
│   │   └── order_provider.dart
│   └── auth/
└── shared/
    └── widgets/
```
