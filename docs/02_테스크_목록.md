## 1. 진행 방식(규칙)

### 1.1 작업 단위(티켓/커밋) 기준

- **1티켓 = 1가치**(화면 1개, API 1개, 마이그레이션 1개 등)
- PR(또는 커밋)에는 **테스트 방법**을 같이 적는다(수동 포함)
- DB/환경 비밀값은 **절대 커밋 금지**

### 1.2 “Done(완료)” 정의

아래 조건을 모두 만족하면 완료로 본다.

- [ ] 동작 확인(로컬에서 재현 가능)
- [ ] 로그로 오류 원인 추적 가능
- [ ] 문서(이 파일 또는 관련 문서) 업데이트
- [ ] 병행 운영 시 “대조”가 가능한 지표가 존재
