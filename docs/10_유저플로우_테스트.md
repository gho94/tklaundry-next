# 유저플로우 테스트 — 레거시 동등 검증

> **목적**: 차세대 앱을 **절차대로** 켜 보며, 같은 업무를 레거시(C#)에서 했을 때와 **결과가 같은지** 수동 확인한다.  
> **기준**: `00_레거시_분석.md`, 화면 구조 `08_IA_정보구조.md`  
> **사용법**: API·앱 기동 후 아래 순서대로 진행. 각 단계 끝에 `통과 / 실패 / 미구현`만 표시한다.

### 실행 전

| 항목 | 내용 |
|------|------|
| DB | 레거시와 동일 `TKLaundry` (또는 동일 복제본) |
| 레거시 | `TKLaundry.exe` — 비교할 화면만 열어 둠 |
| 차세대 | API `bootRun` + `flutter run -d windows` |
| 테스트 계정 | `ComMember`에 `UseYn='Y'` 계정 1개 (아이디·비번 기록) |

### 결과 표기

- **통과**: 레거시와 동일하게 동작 (또는 레거시에 없는 기능이면 PRD/IA대로 동작)
- **실패**: 다름 — 간단히 메모
- **미구현**: 화면·API 없음 — 나중에 재실행

---

## 1. 앱 시작 → 로그인

레거시: 실행 시 `FrmLogIn` → 성공 시 메인(접수 목록).

| # | 절차 (차세대) | 레거시와 같아야 할 것 | 결과 |
|---|--------------|----------------------|:----:|
| 1-1 | 앱 실행 | 로그인 창이 **먼저** 나온다 (메인 바로 X) | 통과 |
| 1-2 | 로그인 화면 확인 | 아이디·비밀번호 입력란, 로그인 버튼 | 통과 |
| 1-3 | 빈 값으로 [로그인] | 진입 안 됨, 안내 메시지 | 통과 |
| 1-4 | 틀린 비밀번호로 [로그인] | 메인 안 됨, 오류 표시 | 통과 |
| 1-5 | 맞는 계정으로 [로그인] | 메인(접수) 화면으로 이동 | 통과 |
| 1-6 | 상단에 로그인한 **사용자명** 표시 | 레거시와 동일 계정명 수준 | 통과 |
| 1-7 | 앱 종료 후 재실행 | 다시 로그인 화면 (세션 정책에 맞게) | 통과 |

## 2. 회원가입 (차세대에만 있는 흐름)

레거시: 운영 중 회원가입 UI는 보통 없음. **DB `ComMember` 등록**과 동일하게 쌓이는지만 보면 된다.

| # | 절차 (차세대) | 레거시와 같아야 할 것 | 결과 |
|---|--------------|----------------------|:----:|
| 2-1 | 로그인 화면 → [회원가입] | 회원가입 화면 진입 | 통과 |
| 2-2 | 아이디 입력 → [중복 확인] | 이미 있는 ID면 사용 불가 | 통과 |
| 2-3 | 새 아이디·이름·비번·비번확인 입력 후 가입 | 성공 안내 | 통과 |
| 2-4 | DB `ComMember` 조회 | `UseYn='Y'`, 방금 ID 존재 | 통과 |
| 2-5 | 방금 ID로 로그인 | 1-5와 같이 메인 진입 | 통과 |
| 2-6 | 비밀번호 불일치·필수 누락 시 가입 | 저장 안 됨, 안내 | 통과 |

## 3. 로그인 후 메인(접수 목록)

레거시: 접수 목록 = 하루 업무 시작 화면.

| # | 절차 (차세대) | 레거시와 같아야 할 것 | 결과 |
|---|--------------|----------------------|:----:|
| 3-1 | 로그인 직후 화면 | **접수 목록**이 기본 (다른 메뉴 X) | |
| 3-2 | 좌측 메뉴 | 「접수」가 있고 선택됨 | |
| 3-3 | 기간 기본값 | **오늘** 접수가 보이게 (또는 레거시와 동일 기본) | |
| 3-4 | 목록 컬럼 | 접수번호·일시·고객·상태·금액 등 핵심 열 | |
| 3-5 | **총 건수·총액** (합계) | 같은 기간·필터에서 레거시 목록 합계와 **숫자 일치** | |
| 3-6 | 기간을 어제/7일 등으로 변경 | 건수·목록이 레거시와 같게 변함 | |
| 3-7 | 상태 필터 1종 적용 | 레거시와 같은 건만 남음 | |
| 3-8 | 고객명·전화 검색 | 레거시 검색과 같은 건이 잡힘 | |
| 3-9 | 행 선택 | 상세로 이어짐 (패널 또는 상세 화면) | |

> **3-5가 병행 핵심**: 불일치 시 `05_DB_병행_및_마이그레이션.md` 수기 기록표에 날짜·차이 적기.
